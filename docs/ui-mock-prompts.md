# UI Mock Prompts

このファイルは UI デザイン mock を生成・依頼するときの prompt 台本です。React 実装そのものではなく、画面構成の合意に使います。

## Home / City Overview

Mobile portrait UI for a Japanese tactical city finance defense game named "破産防衛都市". Show a city map as the main surface, top compact resource bar with credit, cash, debt, resident satisfaction, bottom command dock with icons for policy, repair, gacha, mission. Calm civic teal, amber warnings, readable Japanese UI placeholders, no marketing hero, actual playable interface.

## Battle Wave

Mobile portrait battle screen for a city bankruptcy defense strategy game. Central city district grid under financial pressure waves, city hall highlighted, crisis alerts coming from edges, bottom one-thumb command dock, right side compact event queue, high readability, no fantasy enemies, no oversized tutorial text.

## Policy Card Inventory

Mobile card inventory screen for municipal policy cards. Dense but readable list/grid, card art thumbnails, rarity and cooldown indicators, filter tabs, lower action bar. Work-focused game UI, civic colors, no decorative blobs, no landing page style.

## Free Currency Gacha

Mobile gacha screen using free currency only. Reconstruction support package as the reward source, clear daily limit indicator, draw button, probability/info access, Turnstile protected state implied but not visually technical. Avoid casino, slot machine, or gambling-heavy imagery.

## Login / Guest Merge

Mobile account screen for a game supporting guest play and login sync. Show current guest progress, login options, merge explanation space, safe confirmation button, compact status rows. Practical and calm, no scary warning wall.

## Results / Bankruptcy Report

Mobile results screen after a wave. Show city credit rating, debt movement, districts saved, resident satisfaction, rewards, and next recommended action. Use a municipal report visual style but keep it satisfying as a game reward screen.

## Desktop Reference

Responsive desktop layout for the same game, not a landing page. Keep the city map central, resource rail left, event log right, command dock bottom. It should feel like a playable operations room, not a marketing website.
